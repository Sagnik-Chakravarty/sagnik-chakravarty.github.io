## Overview

This repository contains the source code for my personal academic website, hosted via GitHub Pages. The website serves as a centralized platform for presenting my research, projects, and academic trajectory across survey methodology, data science, and computational social science.

I am currently a graduate student in the Joint Program in Survey Methodology (JPSM) at the University of Maryland, College Park, specializing in Survey and Data Science.

---

## Research Focus

My work lies at the intersection of statistical methodology and modern machine learning, with an emphasis on how data is generated, measured, and modeled. In particular, I am interested in:

- **Survey Methodology** — sampling design, Total Survey Error, nonresponse, and measurement error  
- **Computational Social Science** — modeling public discourse and narrative formation  
- **Machine Learning** — data-centric modeling, interpretability, and robustness  
- **Natural Language Processing** — stance detection, framing analysis, and LLM-assisted pipelines  
- **Large Language Models (LLMs)** — integrating statistical rigor into generative systems  

A central theme of my work is that many failures in machine learning systems arise from issues in data generation and measurement rather than model architecture alone.

## Featured Project

### FrameScope: AI Metaphor and Framing Analysis

FrameScope is a multimodal research pipeline for analyzing how artificial intelligence is framed in public discourse.

- LLM-assisted metaphor detection across 19 categories  
- Stance classification and sentiment analysis  
- Integration of Reddit and news data sources  
- Interactive dashboard for temporal and thematic exploration  

This project reflects my broader interest in combining computational methods with survey and communication theory to study public perception.

🔗 **Live Dashboard:** https://framescope.streamlit.app  
🔗 **GitHub Repository:** https://github.com/Sagnik-Chakravarty/FrameScope

## SagnikGPT: Interactive Research Assistant

The website includes an embedded LLM-powered assistant that allows users to query my portfolio interactively.

- Retrieval-Augmented Generation (RAG) pipeline over my resume, research papers, posters, and project documentation  
- Semantic search using vector embeddings  
- Context-aware response generation with citation-style outputs  
- Session-based conversational memory  

This component demonstrates the integration of modern LLM systems with structured academic data.

---

## Technical Stack

The website and assistant are built using:

- **Frontend:** HTML5, CSS3 (custom responsive design), JavaScript  
- **Backend:** FastAPI (for LLM interaction and retrieval pipeline)  
- **LLM Infrastructure:** Groq API (Llama-based models)  
- **Vector Database:** ChromaDB  
- **Data Processing:** Python (PDF, CSV, OCR-based ingestion pipelines)  
- **Deployment:** GitHub Pages (frontend)  

---

## Design Principles

The website is designed with the following priorities:

- **Clarity and Readability** — academic-style layout emphasizing content over decoration  
- **Responsiveness** — optimized for desktop, tablet, and mobile viewing  
- **Interactivity** — project exploration through visual and conversational interfaces  
- **Reproducibility Mindset** — clear separation of data, methods, and outputs
