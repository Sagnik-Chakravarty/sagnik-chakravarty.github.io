How SagnikGPT Was Built

SagnikGPT is a retrieval-augmented portfolio assistant built to answer questions about my resume, research papers, posters, project notes, and work experience. Instead of relying on the model’s general knowledge, the system retrieves relevant chunks from my portfolio knowledge base and uses an LLM to synthesize grounded answers.

1. Knowledge Base

Resume, papers, posters, transcripts, and project notes are parsed from PDF, TXT, CSV, and image files.

2. Retrieval Layer

Documents are chunked, embedded, and stored in a local vector database for semantic search.

3. LLM Backend

A FastAPI backend retrieves relevant context and sends it to a Groq-hosted Llama model for response generation.

4. Web Interface

The assistant is embedded directly into this portfolio with markdown rendering, session memory, and source-aware responses.

Tech stack: HTML/CSS/JavaScript, FastAPI, ChromaDB, Groq, Llama, Python, PDF/CSV/TXT ingestion, OCR-assisted poster parsing.

Research Philosophy

Many failures in machine learning and computational social science are not only model failures; they are failures to understand the data-generating process. My research focuses on building methods that treat data quality, measurement, sampling, and uncertainty as central parts of inference rather than downstream technical details.
